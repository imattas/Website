```c
    win();

```

## Key Takeaways

- Not every "Binary Exploitation" challenge requires memory corruption -- this one is a source-reading exercise that hands you the flag for correct answers.
- The answers live in the C source: `#define` macros, array sizes, function names (`win`/`print_flag`), and constants.
- pwntools (`process`/`remote` + `recvuntil`/`sendline`) makes automating the prompt-response loop trivial once the answers are known.
- A `--read-source` helper that dumps local `.c`/`.h` files speeds up finding the answers.
